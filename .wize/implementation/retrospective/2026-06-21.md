# Retrospective — Sprint security-overlay (AI Pentester)

## Sprint snapshot
- Escopo: 8 epics (E01–E08), 27 stories — **todas done**.
- Cobertura de testes: 246 testes `node:test` verdes; `npm run validate` verde (73 assets).
- Entregue: overlay file-first completo (recon → enumerate → SAST → DAST → report) + preflight/instalador + report executivo.
- Release: **v0.6.0** publicado no npm (Trusted Publishing + provenance) + GitHub Release. Dry-run verde antes da tag.
- Validação real: rodado end-to-end contra `sabia/starterkit-sabia` (Laravel/PHP, app via docker em :8082). Cobertura 100% no run final; risco CRÍTICO (1 crit, 105 high — 97 secrets + 8 dep CVEs, `.htaccess` exposto).
- Carry-over: 0.

## Worked
- **TDD disciplinado por story.** Cada módulo (`scope-gate`, `cvss`, `partial`, `allowlist`, `owasp`) nasceu com teste primeiro; a suíte pegou regressões a cada fix de CLI.
- **Zero-dep como restrição de design.** Forçar Node built-ins manteve o overlay alinhado ao "zero runtime próprio" e evitou supply-chain — e ainda assim entregou CVSS v3.1 e parser YAML próprios.
- **Gate de escopo centralizado (`assertTargetInScope`).** Um único ponto de decisão + `.refusals.log` tornou o comportamento de segurança testável e auditável.
- **Validar contra um alvo real (sabia).** Foi o que revelou TODOS os bugs de integração que os mocks não pegaram. Mock prova a lógica; alvo real prova o produto.
- **AI insights via harness, não API externa.** O renderer consome `ai-insights.json` escrito pelo LLM do harness — honra "dados ficam locais" e ainda entrega briefing executivo + plano P0/P1/P2.

## Didn't work
- **Mocks esconderam incompatibilidade de CLI das ferramentas reais.** gitleaks (`-f` virou formato, exit≠0 com leaks), osv-scanner v2 (`scan source -L`), nuclei v3 (sem `-t passive`), ffuf (wordlist inexistente) — todos passavam nos testes mas falhavam no run real. Os testes mockavam o `execFile` e nunca exercitaram a sintaxe real.
- **`detectTools` sem `cacheDir` explícito.** Os 8 scripts detectavam ferramentas contra o cwd do kit, não o `.wize/security` do alvo — gerou falso "ferramenta ausente".
- **Merge de partials inconsistente.** `mergeSast`/`mergeDast` tinham 3 assinaturas diferentes entre scripts e sobrescreviam o bloco `tools` do frontmatter — quebrou a contagem de cobertura.
- **Wordlist do ffuf era um TODO não cumprido.** Referenciada em E06-S04 mas nunca criada; só apareceu no run real (ffuf crashou).
- **Report inicial confundia superfície com vulnerabilidade.** Portas/endpoints apareciam como "findings" de severidade — corrigido só após feedback de revisão.

## Surprised us
- **gitleaks/nuclei/ffuf/osv-scanner NÃO estão no apt.** O instalador gerado mandava `apt install` em tudo e falharia inteiro num vibecoder real (Mac/WSL). Exigiu reescrever o gerador com fonte-por-ferramenta (apt vs GitHub release vs script).
- **App alvo precisa estar de pé para o DAST.** Óbvio em retrospecto, mas o run "completo" rodou com a app parada e gerou DAST vazio — o report precisou aprender a declarar "app offline / não testável" honestamente.
- **A honestidade de cobertura virou feature.** O pedido "não finja que auditou" gerou a seção de audit-confidence — provavelmente o diferencial mais valioso vs. ferramentas que reportam só o que acharam.

## Changes for next sprint (3 max)

1. **Teste de contrato real por ferramenta (smoke opt-in).** Para cada tool, um teste que roda o binário de verdade (skip se ausente) validando a sintaxe da CLI — pega quebra de versão sem depender de run manual. *Owner: Hawkeye + Shuri. By: próxima sprint, dia 1.*
2. **Centralizar `mergePartial` no `_shared/partial.js`.** Uma única função que mescla seções + tools, usada por todos os scripts SAST/DAST. Mata a classe inteira de bugs de overwrite. *Owner: Tony. By: incluída como story na sprint de correção.*
3. **Auto-sugestão de sprint pós-scan (a feature pedida).** Ao fim do `wize-sec-pentest`, gerar um backlog de correção (epics/stories) a partir dos findings, pronto para `wize-create-epics-and-stories`. *Owner: Pepper + red-teamer. By: planejar agora nesta mesma sessão.*

## Decisions made
- `ai-insights.json` é o contrato oficial entre o LLM do harness e o renderer (sem API externa).
- Cobertura/audit-confidence é parte obrigatória do report (não opcional).
- DAST exige alvo vivo declarado no `scope.md`; sem app, o report marca explicitamente como não testado.
