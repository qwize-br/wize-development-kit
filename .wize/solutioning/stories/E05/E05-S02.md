# Story: Create `wize-checkpoint-preview`

## Context

`wize-dev-story` runs end-to-end. For M/L stories Shuri is exposed: by the time she realizes the design was wrong, half the code is written. This story adds a checkpoint that runs mid-story to validate direction before continuing.

## Acceptance criteria

- **AC-E05-3:** Workflow `src/method-skills/4-implementation/wize-checkpoint-preview/workflow.md` exists with frontmatter `status: ready`.

- Decision is recorded in `.wize/implementation/checkpoints/{story_id}.md`.
- `npm test` remains green.
- Adapters IDE regenerated.
