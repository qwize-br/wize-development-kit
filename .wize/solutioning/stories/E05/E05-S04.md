# Story: Create `wize-qa-generate-e2e-tests`

## Context

`wize-tea-design` writes a test split (unit / integration / e2e counts) but does not generate the actual E2E cases. Hawkeye needs a dedicated workflow to translate ux-design pages + ACs into concrete E2E test cases (Playwright / Detox, depending on profile).

## Acceptance criteria

- **AC-E05-6:** Workflow `src/tea-skills/wize-qa-generate-e2e-tests/workflow.md` exists with frontmatter `status: ready`.
- Workflow reads `.wize/planning/ux/ux-design/{screen}.md` + `.wize/planning/prd.md` ACs and produces `.wize/implementation/tea/e2e-cases/{screen}.md` with concrete test cases (name, steps, selectors, expected outcome).
- Workflow has 4 sections: Map (UX screen → ACs), Cases (generate 3-10 cases per screen), Selectors (derive from design system + story testid), Triage (mark each case as P0/P1/P2).
- `npm test` remains green.
- Adapters IDE regenerated.

## Out of scope

- Generate Playwright code (the user writes the test bodies).
- Auto-pick the framework (Hawkeye proposes; user confirms).
- Visual / a11y E2E (out of scope; covered by separate audit).
