# Story: Create `wize-project-context`

## Context

Every agent (Shuri, Hawkeye, Mantis, Wizer) loads context from different files. When something changes (new AC, ADR, design decision), the change has to propagate by hand to every consumer. This story creates a single canonical `project-context.md` that consolidates brief + PRD + UX + architecture + decisions, and gives other agents a single source.

## Acceptance criteria

- **AC-E05-1:** Workflow `src/method-skills/3-solutioning/wize-project-context/workflow.md` exists with frontmatter `status: ready`.
- **AC-E05-2:** Workflow produces `.wize/knowledge/project-context.md` with 5 sections: Vision, Audience, Tech stack, Key ADRs, Active risks. Each section pulls from a specific source (brief / PRD / architecture / adrs / risk-profile).
- Workflow is registered via module discovery walker.
- `npm test` remains green.
- Adapters IDE regenerated.

## Out of scope

- Auto-rebuild on every change (manual trigger only).
- Synchronize to other IDE contexts (Cursor rules etc.).
