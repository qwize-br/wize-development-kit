# Story: Create `wize-investigate`

## Context

When a test fails or a regression appears, Shuri currently uses `wize-dev-story` or `wize-quick-dev` ad-hoc. There is no structured investigation flow. This story adds a dedicated workflow for Shuri (with Hawkeye on call) to do root-cause analysis, write a report, and propose a fix path.

## Acceptance criteria

- **AC-E05-4:** Workflow `src/method-skills/4-implementation/wize-investigate/workflow.md` exists with frontmatter `status: ready`.
- **AC-E05-5:** Workflow has 5 sections: Frame (one paragraph: what broke, when, who noticed), Reproduce (steps to reproduce), Hypothesize (top 3 root causes ranked by likelihood), Verify (how to confirm each hypothesis), Conclude (chosen root cause + fix path).
- Output: `.wize/implementation/investigations/{YYYY-MM-DD}-{short-slug}.md`.
- `npm test` remains green.
- Adapters IDE regenerated.

## Out of scope

- Auto-fix (only propose the fix).
- Open a story from the investigation (the user triggers `/wize-create-story`).
- Multi-investigation in one run.

## Notes for Shuri

- Owner: wize-agent-dev (Shuri) + wize-agent-test-architect (Hawkeye on call).
- Phase: 4-implementation.

## Notes for Hawkeye

- Test: workflow file exists; frontmatter complete; 5 sections; output path documented.
