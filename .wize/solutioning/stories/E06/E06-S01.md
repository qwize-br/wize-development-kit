# Story: Create `wize-review-edge-case-hunter`

## Context

`wize-code-review` mentions an Edge Case Hunter layer but does not expose it as a callable skill. Reviewers who want a focused edge-case pass (without the full adversarial review) have to invoke the broader workflow. This story carves out Edge Case Hunter as a standalone skill that the user can call directly.

## Acceptance criteria

- **AC-E06-1:** Skill `src/core-skills/wize-review-edge-case-hunter/skill.md` exists with frontmatter `status: ready` and `name: wize-review-edge-case-hunter`.
- Skill takes a code change (diff, file, or story) and returns a structured list of edge cases (input edges, state edges, time/race edges, integration edges).
- Skill has 4 sections: Scope (what to examine), Hunt (per-area enumeration), Rank (likelihood × impact), Hand-off (cases to test or guard).
- `npm test` remains green.
- Adapters IDE regenerated.

## Out of scope

- Auto-write tests for the edge cases (use `wize-qa-generate-e2e-tests` or Hawkeye's normal flow).
- Full code review (use `wize-code-review` for that).
