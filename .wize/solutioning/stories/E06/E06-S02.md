# Story: Create `wize-index-docs`

## Context

The `.wize/` tree grows as artifacts are added. Wizer and `wize-help` scan for files heuristically, but there is no canonical index. New devs have to discover the structure by reading individual files. This story adds a workflow that rebuilds `.wize/knowledge/index.md` from the actual tree, listing every artifact with one-line descriptions.

## Acceptance criteria

- **AC-E06-2:** Skill `src/core-skills/wize-index-docs/skill.md` exists with frontmatter `status: ready` and `name: wize-index-docs`.
- Workflow scans `.wize/` and produces `.wize/knowledge/index.md` with sections: Planning, Solutioning, Implementation, Knowledge, Custom.
- Each artifact gets a one-line description from its frontmatter `description` (or first heading) plus a relative link.
- `npm test` remains green.
- Adapters IDE regenerated.

## Out of scope

- Index IDE-specific folders (`.claude/`, `.cursor/`, etc.).
- Auto-run on every commit (manual trigger; CI step is a follow-up).
