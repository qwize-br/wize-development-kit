# Story: Create `wize-customize`

## Context

The kit supports overrides via `.wize/custom/{agents,skills,workflows}/<code>/customize.toml` but the user has no guided way to discover or create them. Today they have to read `DECISIONS.md` and write the TOML by hand. This story adds a workflow that asks which persona/skill/workflow they want to customize, generates the customize.toml scaffold, and walks them through the override fields.

## Acceptance criteria

- **AC-E06-5:** Skill `src/core-skills/wize-customize/skill.md` exists with frontmatter `status: ready` and `name: wize-customize`.
- **AC-E06-6:** Skill takes a persona/skill/workflow code and produces a customize.toml stub at `.wize/custom/{type}/{code}/customize.toml` with placeholders for `name`, `title`, `persona` (or skill/workflow body), and `tools`.
- Skill is interactive: confirms the override target, asks what to change, and offers to write the stub.
- `npm test` remains green.
- Adapters IDE regenerated.

## Out of scope

- Apply the override automatically (the user must edit and re-run `wize-dev-kit sync`).
- Override at runtime (the override is loaded via the file system; no hot-reload).
