# Story: Update installer brownfield flow to offer mode selection

## Context

When the installer detects a brownfield repo, it currently offers only a headless baseline run. It should let the user pick the documentation mode.

## Acceptance criteria

- **AC-E03-3:** Brownfield install prompts:
  1. Run quick baseline (default, existing behavior)
  2. Run initial_scan
  3. Run full_rescan
  4. Skip documentation for now
- Non-TTY fallback defaults to quick baseline (preserves CI smoke).
