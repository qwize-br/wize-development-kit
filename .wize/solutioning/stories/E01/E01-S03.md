# Story: Implement JSON state file + resume flow

## Context

Long scans need resumability. BMAD tracks progress in `project-scan-report.json`. Wize needs the same, with a JSON Schema.

## Acceptance criteria

- **AC-E01-3:** A JSON Schema at `src/method-skills/1-analysis/wize-document-project/templates/project-scan-report-schema.json` validates the state file.
- `initState(projectRoot, mode, scanLevel)` creates a valid state file under `.wize/knowledge/document-project/project-scan-report.json`.
- `loadState(projectRoot)` reads and validates an existing state file; returns `null` if missing/invalid.
- `updateState(projectRoot, patch)` merges a patch and writes the file atomically.
- Resume flow detects an existing state <24h old and offers to resume; older states are archived to `.wize/knowledge/document-project/_archive/`.

## Out of scope

- UI/CLI resume prompt logic (handled in CLI story).
- Deep-dive target tracking (handled in deep-dive story).

- Use `Date.now()` style helpers; avoid external date libs.
