# Story: Implement deep_dive mode for specific areas

## Context

BMAD's `deep_dive` reads every file in a selected area (folder, file, feature, API group, component group) and produces a detailed implementation-ready doc.

## Acceptance criteria

- **AC-E01-7:** `wize-dev-kit document-project deep_dive --target <path>` produces `deep-dive-{sanitized-name}.md`.
- Target types supported: `folder`, `file`, `feature` (search by name), `api_group`, `component_group`.
- Deep-dive doc includes: file inventory, exports/signatures, dependencies, dependents, key implementation details, patterns, data flow, integration points, testing, TODOs, risks, modification guidance.
- Master `index.md` is updated with a link to the new deep-dive doc.

## Out of scope

- Visual dependency graph rendering (ASCII list is enough).
- Automatic feature detection UI (target is provided via CLI flag).
