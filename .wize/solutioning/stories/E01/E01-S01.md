# Story: Add CLI argument parsing for document-project subcommand/mode

## Context

Currently `wize-dev-kit` has no `document-project` subcommand. We need a CLI entry point that accepts an optional mode (`quick`, `initial_scan`, `full_rescan`, `deep_dive`) and a `--resume` flag.

## Acceptance criteria

- **AC-E01-1:** `wize-dev-kit document-project` defaults to `quick` mode and produces the 6 baseline files.
- `wize-dev-kit document-project initial_scan`, `full_rescan`, `deep_dive` are recognized.
- `wize-dev-kit document-project --resume` reads `project-scan-report.json` and continues.
- Unknown mode prints helpful error and exits with code 1.

## Out of scope

- Actual implementation of non-quick modes (other stories).
- Interactive prompts for mode selection.

## Notes for Shuri

- Extend `tools/installer/wize-cli.js` dispatcher.
- Add a new `tools/installer/commands/document-project.js` module to keep CLI thin.
- Follow existing command pattern (`commands/update.js`, `commands/agent.js`).
