# Story: Implement initial_scan mode with conditional scans

## Context

`initial_scan` is the BMAD-equivalent full documentation pass for a repo that has no prior docs. It detects project type, scans conditionally, and writes docs.

## Acceptance criteria

- **AC-E01-5:** `wize-dev-kit document-project initial_scan [quick|deep|exhaustive]` runs the full flow.
- Quick initial scan: pattern-only, produces `index.md`, `project-overview.md`, `source-tree-analysis.md`.
- Deep initial scan: reads critical directories per project type and produces conditional docs (`api-contracts.md`, `data-models.md`, `component-inventory.md`, `development-guide.md`, `deployment-guide.md`, `contribution-guide.md`) when applicable.
- Exhaustive initial scan: reads all source files in batches.
- Multi-part repos produce per-part docs and `integration-architecture.md`.

## Out of scope

- Resume (already in E01-S03).
- Deep-dive single area (E01-S07).

- Write-as-you-go: each doc written immediately, context purged.
