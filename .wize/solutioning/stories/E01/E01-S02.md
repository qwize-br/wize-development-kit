# Story: Create project-type classification CSV and classifier module

## Context

BMAD uses `documentation-requirements.csv` to detect project type and decide what to document. Wize needs an equivalent 12-type CSV plus a pure Node module that classifies a repo root.

## Acceptance criteria

- **AC-E01-2:** A CSV at `src/method-skills/1-analysis/wize-document-project/documentation-requirements.csv` exists with 12 rows (web, mobile, backend, cli, library, desktop, game, data, extension, infra, embedded) and columns for detection patterns, scan flags, critical directories, test/config/auth/CI patterns.
- A `classifyProject(rootDir)` function returns `{ projectTypes: [...], parts: [...] }`.
- Classifier correctly identifies `wize-dev-kit` itself as `cli` + `library`.
- Multi-part repos (e.g. `client/` + `server/`) are detected as two parts.

## Out of scope

- Reading source files; this is pattern-only detection.
- Architecture registry/template matching.

## Notes for Shuri

- Module path suggestion: `tools/installer/document-project/classify.js`.
- Keep CSV parsing simple (`fs.readFileSync` + split) or reuse no external deps.
- Add `documentation-requirements.csv` to `files` array in `package.json` if it needs to ship.

## Notes for Hawkeye

- Test classifier against sample directory trees created in temp dirs.
- Assert exact project type for the current repo.
