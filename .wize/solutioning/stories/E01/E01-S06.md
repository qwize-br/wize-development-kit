# Story: Implement batch scanner and full_rescan mode

## Context

Deep and exhaustive scans must avoid loading entire repos into context. BMAD batches by subfolder. Wize needs the same.

## Acceptance criteria

- **AC-E01-6:** A `batchScanner(rootDir, options)` module scans one subfolder at a time, skipping `node_modules`, `.git`, `dist`, `build`, `coverage`.
- Files >5000 LOC are flagged but not fully read in one shot.
- Scanner returns a summary per batch: path, file count, key findings.
- `full_rescan` mode archives the previous `project-scan-report.json` and re-runs `initial_scan` from scratch.

## Out of scope

- File content parsing beyond simple extraction (signatures, exports) — can be shallow in this story.
