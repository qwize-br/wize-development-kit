# Story: Implementar `wize-onboarding` real

## Context

`src/orchestrator-skills/wize-onboarding/workflow.md` é um stub de 28 linhas (`status: stub`). Após `npx wize-dev-kit install`, o primeiro comando do usuário é `/wize-onboarding`, mas hoje ele não faz nada além de sugerir o próximo agente. Esta story substitui o stub por um workflow de triagem real.

## Acceptance criteria

- **AC-E04-1:** Workflow detecta brownfield lendo `.wize/config/project.toml` e sinais de repo (src/, package.json).
- **AC-E04-2:** Se existe `.wize/implementation/sprint-status.yaml`, oferece resumir status e retomar sprint atual.
- **AC-E04-3:** Se não existe brief, oferece iniciar `/wize-product-brief`. Se existe brief sem PRD, oferece `/wize-create-prd`. Se PRD existe, oferece `/wize-sprint-planning`.
- Workflow marca `status: ready` no frontmatter (não mais `stub`).
- `npm test` permanece verde.

## Out of scope

- Mudar `install` para chamar onboarding automaticamente.
- Coletar objetivo em 30 dias (será história separada se virar padrão).
